# 研究生周报生成器

帮助用户生成标准格式的研究生周报，并支持将 Markdown 转换为 PDF。

## 周报格式模板

```
**汇报人**：{姓名}  **汇报周期**：{起始日期}-{结束日期}

### 本周已完成工作
{编号}. {工作内容}
{编号}. {工作内容}
...

### 后续计划与预期
{编号}. {计划内容}
{编号}. {计划内容}
...
```

## 执行流程

1. **询问用户信息**（如未直接提供）：
   - 汇报人姓名
   - 汇报周期（如"3月24日-3月30日"）
   - 本周已完成的工作内容（列表形式）
   - 下周/后续计划（列表形式）
   - 是否需要导出为 PDF

2. **生成周报**：
   - 按照上述格式模板生成 Markdown 格式的周报
   - 确保格式与示例完全一致
   - 日期格式统一为：YYYY年M月D日

3. **保存 Markdown 文件**：
   - 询问用户保存路径
   - 文件名建议格式：`毕业论文周报（{起始日期}-{结束日期}）.md`
   - 将 Markdown 内容写入文件

4. **PDF 导出（如需）**：
   - 使用本 skill 自带的 `scripts/md_to_pdf.py` Python 脚本将 Markdown 转换为 PDF
   - 首先检查所需依赖是否已安装（推荐使用 markdown-pdf）
   - 如未安装，指导用户运行：`pip install markdown-pdf`
   - 运行脚本：`python /Users/zly/.claude/skills/weekly-report/scripts/md_to_pdf.py <输入.md> <输出.pdf>`
   - PDF 文件命名建议：`毕业论文周报（{起始日期}-{结束日期}）.pdf`

## 注意事项

- 保持简洁，每条工作内容一句话描述
- 已完成工作和后续计划分别用编号列表
- 日期范围使用中文括号包裹
- 如有用户未提供的信息，主动询问
- 确保用户已安装 Python 及所需依赖（推荐安装 markdown-pdf: `pip install markdown-pdf`）
- 使用绝对路径调用 `md_to_pdf.py` 脚本
- 脚本支持多种转换方式，会自动选择最佳可用方式
